 React 的 3 个核心思想：
组件化：将复杂界面拆分为独立、可复用的组件，降低维护成本；
单向数据流：数据从父组件通过props流向子组件，子组件不直接修改props，确保数据可追溯；
状态驱动 UI：UI 是状态的 “映射”，只需修改状态（如setValue('X')），React 自动更新 UI，无需手动操作 DOM。


遵循 “单一职责”：一个组件只负责一个功能（如Square只管单个方块的显示与点击，Board管 9 个方块的布局与游戏逻辑）
